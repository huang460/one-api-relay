# 小米「百亿 Token 计划」申请材料

## 项目名称
One API Relay — AI 大模型 API 统一管理与分发平台

## 一、平台效果描述

### 1.1 平台定位
One API Relay 是一个轻量级的 **AI API 统一管理与分发平台**，作为用户与各大 AI 厂商之间的智能中转层，解决多模型、多 Key、多用户的统一管理与调用问题。

### 1.2 核心能力
- **统一接入**: 兼容 OpenAI SDK 格式，一套接口对接 30+ AI 厂商
- **智能调度**: 多 Key 自动轮换，负载均衡，故障自动切换
- **用量管控**: 实时 Token 统计，用户级配额限制，防止滥用
- **安全鉴权**: 多用户管理，独立访问令牌，操作日志审计
- **灵活部署**: 单文件运行，支持 Docker / 二进制部署，内网穿透公网访问

### 1.3 平台效果

| 指标 | 效果 |
|------|------|
| 接入厂商 | 30+（OpenAI / Claude / Google / 百度/ 阿里 / 腾讯 / DeepSeek 等） |
| 兼容性 | 一行代码改 base_url 即可接入 |
| 部署耗时 | 从下载到运行 < 1 分钟 |
| 资源占用 | ~30MB 内存，无外部依赖 |
| 用户管理 | 多用户独立 Key + 配额 |

### 1.4 使用场景
- 团队协作共享 API Key，避免 Key 泄露
- 多模型负载均衡，提高可用性
- Token 用量统计与成本分摊
- 国内环境访问海外 AI API 的统一出口

## 二、项目描述

### 2.1 技术架构

```
┌─────────────┐    ┌──────────────┐    ┌──────────────┐
│  应用 / 用户  │───▶│  One API     │───▶│  OpenAI       │
│  (SDK 接入)   │    │  Relay 平台   │    │  Claude       │
│              │    │              │    │  Google       │
│              │    │  Key 池轮换   │    │  DeepSeek     │
│              │    │  负载均衡     │    │  文心 / 通义   │
│              │    │  Token 统计   │    │  ... 30+ 厂商  │
└─────────────┘    └──────────────┘    └──────────────┘
```

### 2.2 技术栈
- **核心引擎**: One API v0.6.10（Go 语言）
- **数据存储**: SQLite（内置，零配置）
- **公网访问**: ngrok 隧道
- **部署环境**: Windows / Linux / Docker

### 2.3 Agent 开发平台集成
本平台专为 AI Agent 开发场景设计，提供：
- Agent 系统所需的稳定 API 出口
- 多模型按需调度（文本 / 视觉 / 代码生成等）
- 用量监控与成本优化
- 为 Agent 应用提供统一鉴权层

## 三、为什么选择本平台申请资源

1. **轻量高效**: 单文件 35MB，开箱即用
2. **厂商无关**: 一套接口对接所有主流模型
3. **零运维**: 无需数据库、无需复杂配置
4. **成本透明**: 精确 Token 统计，每一分钱都可追溯
